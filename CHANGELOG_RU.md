# Список изменений

### **`Unreleased`**
- **Обновление модели ИИ**: Одиночная генерация 3D переведена с `fal-ai/trellis` на `fal-ai/trellis-2`. Поток одного изображения вызывает `fal-ai/trellis-2` с полем `image_url` и читает URL результата из `model_glb.url` (в Trellis-2 выходное поле называется `model_glb`, в Trellis v1 — `model_mesh`).
- **Мульти-режим с фолбэком**: Мульти-загрузка сначала пробует `fal-ai/trellis-2/multi` со входным `image_urls`. Если этот эндпоинт недоступен (HTTP 404 / Not Found), автоматически выполняется фолбэк на устаревший `fal-ai/trellis/multi`, а URL результата читается из `model_mesh.url`. Срабатывание фолбэка логируется на уровне `WARN` с полным контекстом ошибки fal.ai.
- **Подробное логирование ошибок fal.ai**: Ошибки `ApiError`/`ValidationError` из `@fal-ai/client` теперь логируются вместе со `status`, `requestId`, телом ответа (`body`) и `fieldErrors`, вместо обобщённого `"fetch failed"`. HTTP-ответ клиенту остаётся прежним — общий 500 с верхнеуровневым сообщением.
- **Готовность к деплою**: Бэкенд подготовлен к развёртыванию на [Render](https://render.com). Конфигурация теперь читается из переменных окружения (`HOST`, `PORT`, `DATA_DIR`, `DB_FILE`, `FAL_API_KEY`), а `config.toml` остаётся запасным вариантом для локальной разработки.
- **Постоянное хранилище**: SQLite-база и папки с загруженными/сгенерированными изображениями располагаются внутри `DATA_DIR`, что совместимо с подключаемыми persistent disk'ами.
- **Сборочный пайплайн**: Добавлены npm-скрипты `build` (`tsc`) и `serve` (`node dist/main.js`); сервер по умолчанию слушает `0.0.0.0` и принимает любой порт из диапазона 1–65535.
- **Воспроизводимая установка**: Закоммичен `package-lock.json` и удалён из `.gitignore`.
- **Качество кода**: Исправлены опечатки и проведены мелкие чистки в API-обработчиках, инициализации приложения и общих типах.

### **`v.0.3.0` от 12.04.2026** (Последний релиз)
- **Генерация 3D через ИИ**: Интегрирована модель `fal-ai/trellis` для создания высококачественных 3D-моделей из одного или нескольких фото.
- **Обновленная документация**: Полностью переписан README с подробными спецификациями API, структурой проекта и руководством по настройке.
- **Управление профилем**: Добавлен эндпоинт `/api/v1/profile` для получения данных пользователя.
- **Безопасность**: Реализована аутентификация через Bearer-токен для доступа к профилю.
- **Исправления ошибок**: Устранены ошибки в работе с базой данных и обработке исключений.
- **Производительность**: Оптимизирована загрузка изображений и управление локальным хранилищем.
