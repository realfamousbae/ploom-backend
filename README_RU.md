# **ploom-backend**

Серверная часть проекта Ploom, предоставляющая API для управления пользователями и генерации 3D-моделей на основе ИИ из изображений.

## 🚀 Обзор

**ploom-backend** — это приложение на Node.js, написанное на TypeScript, которое интегрирует передовые модели ИИ для преобразования 2D-изображений в 3D-объекты. Оно обеспечивает аутентификацию пользователей, управление профилями и удобный интерфейс для работы с моделями Trellis от [fal.ai](https://fal.ai).

### Основные возможности
- **Генерация 3D через ИИ**: Создание высококачественных 3D-моделей из одного или нескольких изображений.
- **Управление пользователями**: Безопасная регистрация и авторизация.
- **Обработка изображений**: Загрузка аватаров и наборов изображений для генерации с помощью Multer.
- **Постоянное хранение данных**: Использование SQLite для быстрого и надежного управления данными.
- **Разработка на TypeScript**: Полностью типизированная кодовая база для удобства поддержки и разработки.

## ⚙️ Конфигурация

Проект поддерживает два источника конфигурации: переменные окружения имеют приоритет над TOML-файлом. Это делает его одинаково удобным как для локальной разработки, так и для деплоя на хостинги вроде Render.

### Переменные окружения

| Переменная | Описание | По умолчанию |
|:-----------|:---------|:-------------|
| `HOST` | Хост, на котором стартует HTTP-сервер. | `0.0.0.0` |
| `PORT` | Порт (1–65535). На Render задаётся автоматически. | `3000` |
| `DATA_DIR` | Корневая директория для записываемых данных (БД и изображения). На проде — путь монтирования постоянного диска. | `./` |
| `DB_FILE` | Путь к файлу SQLite. Если относительный — резолвится внутри `DATA_DIR`. | `db/main.sqlite` |
| `FAL_API_KEY` | API-ключ для [fal.ai](https://fal.ai). **Обязателен.** | — |

Если задана `FAL_API_KEY`, сервер запускается без `config.toml`. В противном случае при первом запуске будет сгенерирован шаблон `config.toml` и процесс завершится — заполните файл вручную.

### `config.toml` (запасной вариант для локальной разработки)

```toml
[server]
hostname = "localhost"
port = 3000

[database]
file = "db/main.sqlite"
schema = "db/schema.sql"

[api]
key = "vash-fal-ai-api-key" # Получите на https://fal.ai
```

При запуске приложение автоматически создаёт необходимые директории (`DATA_DIR/public/profile_images`, `uploaded_images`, `generated_images`, а также директорию для файла SQLite).

## 📡 Документация API

### Список эндпоинтов

| Путь | Метод | Описание |
|:-----|:------:|:------------|
| `/` | `GET` | Информация об API и проверка работоспособности |
| `/api/v1/authorize-user` | `POST` | Авторизация пользователя |
| `/api/v1/register-new-user` | `POST` | Регистрация нового аккаунта |
| `/api/v1/profile` | `GET` | Получение данных профиля |
| `/api/v1/generate-from-single` | `POST` | Генерация 3D из одного фото |
| `/api/v1/generate-from-multiple` | `POST` | Генерация 3D из 1-5 фото |

---

### Подробные спецификации

#### 1. **Авторизация пользователя**
`POST /api/v1/authorize-user`
- **Параметры запроса (Query)**:
  - `email` (string): Email пользователя.
  - `password` (string): Пароль.
- **Успешный ответ**: `200 OK`
  ```json
  {
    "message": "User successfully authorized. All data is correct.",
    "token": "...",
    "user": { "name": "...", "surname": "...", "email": "..." }
  }
  ```

#### 2. **Регистрация пользователя**
`POST /api/v1/register-new-user`
- **Параметры запроса (Query)**: `name`, `surname`, `email`, `password`.
- **Данные формы (Body)**:
  - `profile_image` (файл, опционально): Аватар пользователя.
- **Успешный ответ**: `200 OK`
  ```json
  { "message": "User successfully registered.", "token": "..." }
  ```

#### 3. **Профиль пользователя**
`GET /api/v1/profile`
- **Заголовки**:
  - `Authorization`: `Bearer <token>`
- **Успешный ответ**: `200 OK`
  ```json
  { "name": "...", "surname": "...", "email": "...", "profile_image_path": "..." }
  ```

#### 4. **Генерация ИИ (одно изображение)**
`POST /api/v1/generate-from-single`

Использует модель `fal-ai/trellis-2`.
- **Параметры запроса (Query)**:
  - `user_id` (number): ID пользователя.
- **Данные формы (Body)**:
  - `image` (файл): Исходное изображение.
- **Успешный ответ**: `200 OK` — `generated_image_url` содержит ссылку на сгенерированную модель `.glb`.
  ```json
  {
    "message": "Image generated successfully.",
    "generated_image_url": "https://....glb"
  }
  ```

#### 5. **Генерация ИИ (несколько изображений)**
`POST /api/v1/generate-from-multiple`
- **Параметры запроса (Query)**:
  - `user_id` (number): ID пользователя.
- **Данные формы (Body)**:
  - `images` (файлы[]): Массив от 1 до 5 изображений.
- **Успешный ответ**: `200 OK`
  ```json
  {
    "message": "3D Model generated successfully.",
    "generated_image_url": "https://..."
  }
  ```

## 🛠 Установка и запуск

1. **Установка зависимостей**:
   ```bash
   npm install
   ```
   В репозитории есть `package-lock.json` для воспроизводимой установки.

2. **Настройка конфигурации**:
   Либо задайте переменные окружения (как минимум `FAL_API_KEY`), либо отредактируйте сгенерированный `config.toml`.

3. **Запуск (Разработка)**:
   ```bash
   npm start          # ts-node, без сборки
   ```

4. **Сборка и запуск (Production)**:
   ```bash
   npm run build      # tsc -> dist/
   npm run serve      # node dist/main.js
   # или одной командой:
   npm run js
   ```

## ☁️ Деплой (Render)

Бэкенд готов к развёртыванию на [Render](https://render.com) как Web Service.

- **Build Command**: `npm install && npm run build`
- **Start Command**: `npm run serve`
- **Переменные окружения**: задайте `FAL_API_KEY`. При необходимости переопределите `HOST`, `DB_FILE` и т.д. `PORT` Render подставляет автоматически.
- **Постоянный диск (Persistent Disk)**: подключите диск и укажите `DATA_DIR` равным точке монтирования (например, `/var/data`). Тогда SQLite-база и загруженные/сгенерированные изображения будут сохраняться между деплоями.

По умолчанию сервер слушает на `0.0.0.0` и принимает любой порт из диапазона 1–65535, поэтому никаких дополнительных настроек для сетевой части Render не нужно.
